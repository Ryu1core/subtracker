# Subtracker API — документация для фронтенда

Бэкенд: Spring Boot, запускается на `http://localhost:8080`.
Все эндпоинты возвращают JSON. Все суммы — числа с десятичной точкой (например `649.00`).

## Модель подписки

Так выглядит подписка в ответах API:
{
"id": 1,
"name": "Netflix",
"price": 649.00,
"billingDate": "2026-07-15",
"category": "video",
"billingCycle": "MONTHLY",
"createdAt": "2026-07-09T02:24:55.431287"
}


| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Генерируется сервером, при создании не передавать |
| `name` | string | Название подписки |
| `price` | number | Цена за один платёж (не обязательно за месяц!) |
| `billingDate` | string | Дата списания, формат `YYYY-MM-DD` |
| `category` | string | Категория (свободный текст: `video`, `music`, ...) |
| `billingCycle` | string | Период оплаты, см. ниже |
| `createdAt` | string | Когда создана запись, ставит сервер |

### billingCycle — возможные значения (для менюшки-селекта)

- `WEEKLY` — раз в неделю
- `MONTHLY` — раз в месяц (значение по умолчанию)
- `QUARTERLY` — раз в 3 месяца
- `YEARLY` — раз в год

Присылать строго БОЛЬШИМИ буквами, как выше.

## Эндпоинты

### 1. Список всех подписок

GET /api/subscriptions

Ответ `200`: массив подписок (см. модель выше). Если подписок нет — пустой массив `[]`.

### 2. Создать подписку


POST /api/subscriptions
Content-Type: application/json

Тело запроса:

​
{
"name": "Netflix",
"price": 649.00,
"billingDate": "2026-07-15",
"category": "video",
"billingCycle": "MONTHLY"
}

Правила валидации:
- `name` — обязательное, не пустое
- `price` — обязательное, строго больше нуля
- `billingCycle` — одно из четырёх значений выше; если не прислать, будет `MONTHLY`

Ответ `201`/`200`: созданная подписка целиком (с `id` и `createdAt`).
Ответ `400`: если валидация не прошла (пустое 

### 3. Изменит
PUT /api/subscriptions/{id}
Content-Type: application/json

Тело — как у POST (все поля целиком, не только изменённые).

Ответ `200`: обновлённая подписка.
Ответ `404`: подписки с таким `id` нет.
Ответ `400`: валидация не прошла.

### 4. Удалить подписку

​
DELETE /api/subscriptions/{id}

Ответ `200`/`204`: удалено.
Ответ `404`: подписки с таким `id` нет.

### 5. Сумма расходов в месяц

​
GET /api/subscriptions/total

Возвращает общий расход в месяц, где все подписки приведены к месячной цене:
годовая 1200 считается как 100/мес, недельная умножается на ~4.33 и т.д.

Пример: подписки 500/мес + 1200/год → ответ `600`.

## Ошибки

- `400 Bad Request` — невалидные данные (описание ошибки в теле ответа)
- `404 Not Found` — подписка с указанным id не существует

## Как запустить бэкенд локально

1. Установить PostgreSQL 17, создать базу `subtrack`
2. Скопировать настройки из `src/main/resources/application.properties`,
   подставить свой пароль от Postgres
3. Запустить: `./gradlew bootRun` (Windows: `.\gradlew.bat bootRun`)
4. API доступно на `http://localhost:8080`

## Авторизация

API закрыто: все эндпоинты подписок требуют токен. Без токена — `401 Unauthorized`.

### Флоу для фронтенда

1. Юзер регистрируется (один раз) → `POST /api/auth/register`
2. Юзер логинится → `POST /api/auth/login` → в ответе приходит **токен**
3. Сохрани токен (например, в `localStorage`)
4. Прикладывай его к **каждому** запросу к `/api/subscriptions/**` заголовком:
   ​
   Authorization: Bearer <токен>
5. Токен живёт **24 часа**. Поймал `401` — токен протух или невалиден → отправляй юзера на экран логина

### POST /api/auth/register — регистрация

Запрос:
​
{
"email": "user@mail.ru",
"password": "secret123"
}

Правила: email — валидный адрес, пароль — минимум 6 символов.

Ответы:
- `201 Created` — успех:
  ​
  {
  "id": 1,
  "email": "user@mail.ru",
  "createdAt": "2026-07-10T14:43:38"
  }
- `409 Conflict` — email уже занят
- `400 Bad Request` — невалидные данные (в теле будет текст, что именно не так — можно показывать юзеру)

### POST /api/auth/login — вход

Запрос:
​
{
"email": "user@mail.ru",
"password": "secret123"
}

Ответы:
- `200 OK` — успех:
  ​
  {
  "token": "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJ0ZXN0QG1haWwucnUi..."
  }
- `401 Unauthorized` — неверный email или пароль (специально не уточняем, что именно, — так безопаснее)

### Пример запроса с токеном

​
GET /api/subscriptions
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJ0ZXN0QG1haWwucnUi...
